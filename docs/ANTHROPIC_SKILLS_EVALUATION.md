### 2. Frontend Design ⭐⭐⭐⭐⭐

**技能 ID**: `frontend-design`
**評分**: 5/5（強烈推薦）

| 項目 | 內容 |
|------|------|
| **核心功能** | 創建具有獨特性的生產級前端界面 |
| **主要用途** | - 高品質 UI/UX 設計<br>- React/Vue 組件開發<br>- 響應式布局設計<br>- 視覺美學提升 |
| **適用範圍** | Web 組件、頁面、儀表板、React 組件、HTML/CSS 布局 |
| **觸發關鍵字** | 前端設計、UI 設計、介面美化、React 組件、網頁設計 |

**核心設計原則**:

1. **戰略方向**（編碼前確定）

   ```
   選擇明確的美學風格:
   - Brutalist（粗獷主義）
   - Maximalist（極繁主義）
   - Minimalist（極簡主義）
   - Retro-futuristic（復古未來主義）
   - Luxury（奢華風格）
   - Playful（趣味風格）

   原則: 重視「意圖性」而非「強度」
   ```

2. **核心元素**

   | 元素 | 最佳實踐 |
   |------|---------|
   | **排版** | 選擇美麗、獨特、有趣的字體，避免通用預設 |
   | **色彩主題** | 使用 CSS 變數建立連貫調色板，主導色 + 鮮明強調色 |
   | **動效** | CSS 動畫 + 滾動觸發效果，關鍵時刻增強衝擊力 |
   | **空間構圖** | 非對稱、重疊、打破網格的布局 |
   | **視覺細節** | 漸層、紋理、自訂效果增加情境深度 |

3. **避免通用 AI 美學**

   ```
   ❌ 禁止:
   - 過度使用的字體
   - 可預測的紫色漸層
   - Cookie-cutter 布局
   - 模板化設計模式

   ✅ 目標:
   - 情境特定的設計
   - 創意差異化
   - 令人難忘的界面
   ```

**適用場景**:

- ✅ 網站和 Landing Page 設計
- ✅ 儀表板和數據可視化介面
- ✅ 互動式 React/Vue 組件
- ✅ 海報和宣傳 UI 設計
- ✅ 任何需要精緻、難忘前端介面的場景

**適用性評估**:

| 項目 | 評分 | 說明 |

| **實用性** | ⭐⭐⭐⭐⭐ | 前端開發必備技能 |

### 3. Web Artifacts Builder ⭐⭐⭐⭐

### 4. Webapp Testing ⭐⭐⭐

**技能 ID**: `webapp-testing`
**評分**: 3/5（可參考）

| 項目 | 內容 |
|------|------|
| **核心功能** | 基於 Playwright 的 Web 應用自動化測試 |
| **主要用途** | - 驗證前端功能<br>- 調試 UI 行為<br>- 截圖測試<br>- 瀏覽器日誌查看 |
| **支援測試** | 靜態 HTML + 動態 Web 應用 |

**測試類型**:

1. **靜態 HTML 測試**

   ```
   - 直接讀取 HTML 檔案
   - 識別選擇器
   - 無需伺服器
   ```

2. **動態網頁應用測試**

   ```
   - 需要伺服器運行
   - 涵蓋複雜互動場景
   - 完整的端到端測試
   ```

**使用方式**:

```bash
# 單一伺服器啟動
python scripts/with_server.py \
  --server "npm run dev" \
  --port 5173 \
  -- python your_automation.py
```

**測試模式**: 偵察-然後-行動

```
1. 等待網頁完全加載（networkidle）
2. 透過截圖或 DOM 檢查識別選擇器
3. 執行測試操作
```

**關鍵實踐**:

- ✅ 等待 `networkidle` 後再檢查 DOM（動態應用必須）
- ✅ 將 `scripts/` 工具當作黑盒使用
- ✅ 避免不必要的代碼閱讀

**適用性評估**:

| 項目 | 評分 | 說明 |
|------|------|------|
| **實用性** | ⭐⭐⭐ | E2E 測試有用，但工具已成熟 |
| **學習曲線** | ⭐⭐⭐ | Playwright 本身易學 |
| **即用性** | ⭐⭐⭐ | 需要配置環境 |
| **獨特性** | ⭐⭐ | 市面上類似工具很多 |

**推薦理由**:

- ✅ 官方認證的測試最佳實踐
- ✅ 基於成熟的 Playwright 框架
- ✅ 適合 Claude Code 整合

**不推薦理由**:

- ⚠️ Playwright 官方文檔已很完善
- ⚠️ 沒有顯著的獨特優勢
- ⚠️ 可能增加不必要的抽象層

**建議行動**:

- 📖 **先參考**: 了解測試最佳實踐
- 🔧 **選擇性採用**: 根據專案需求決定
- 💡 **直接使用 Playwright**: 如果團隊已熟悉
